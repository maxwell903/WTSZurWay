## How to use this bundle

### One-time setup (you do this manually before Sprint 0 even starts)

1. Create your project repo. The folder you mentioned (`WTSZurWay`) is fine. Initialize it:
   ```bash
   cd WTSZurWay
   git init
   ```
2. Copy the **root files** into `WTSZurWay/`:
   - `CLAUDE.md` (the Master CLAUDE.md from this bundle)
   - `DECISIONS.md` (the empty append-only log)
3. Copy `PROJECT_SPEC.md` (from the project Knowledge in your Anthropic project) into `WTSZurWay/PROJECT_SPEC.md`. The Master CLAUDE.md and every sprint reference it.
4. Read `SPRINT_SCHEDULE.md` once, end to end, so you understand what is happening when.

### For every sprint

1. Open the sprint folder, e.g. `sprints/sprint-00-foundation/`.
2. Read `SPRINT.md`. Confirm you agree with the plan.
3. Copy `CLAUDE.md` from that sprint folder into the working directory of your Claude Code session — overwrite or coexist with your repo's root `CLAUDE.md`. Recommended approach: keep the root `CLAUDE.md` untouched and create a per-sprint `CLAUDE_SPRINT.md` next to it. Both are read by Claude Code.
   - Simpler alternative: replace the root `CLAUDE.md` with the sprint's CLAUDE.md for the duration of that sprint, and restore the master afterwards. Either works.
4. Open a fresh Claude Code session in that working directory.
5. Paste the contents of `PROMPT.md` as the very first message.
6. Let Claude Code execute. Respond to any **Deviation Reports** as instructed in §4 of the Master CLAUDE.md.
7. When you receive a **Sprint Completion Report**, verify the "External Actions Required" block and do whatever is listed (Vercel, Supabase, API keys, etc.).
8. Commit the sprint, run the full test suite one more time, and move to the next sprint.

### Running sprints in parallel

The dependency graph in `SPRINT_SCHEDULE.md` shows which sprints can run in parallel. When two sprints can run in parallel, use **git worktrees**:

```bash
git worktree add ../WTSZurWay-sprint-3 sprint/03
git worktree add ../WTSZurWay-sprint-4 sprint/04
```

One Claude Code session per worktree. Merge in **planned order**, not finish order. Run the full test suite after every merge.

## What every sprint will tell you to do, every time

Every sprint's prompt instructs Claude Code to end with a **Sprint Completion Report** that includes an **External Actions Required** block. That block will tell you, in plain English:

- Whether you need to do anything in **Vercel** (create a project, set an env var, etc.).
- Whether you need to do anything in **Supabase** (create a bucket, run a migration, etc.).
- Whether you need to **install or configure anything locally** (Node, pnpm, Docker, Supabase CLI).
- Whether you need to **obtain or rotate any API keys**.

If the block says "None", you are clear to commit and move on.
